Release Deliverables

The following deliverables are required for the Eclipse-RPM 3.0 release:

  • Source code tagged as eclipse-rpm-3_0 in the Eclipse-RPM repository
  • Eclipse-RPM binaries (downloadable)
Release Milestones/Timeline

Eclipse-RPM 3.0 development will mirror CDT 3.0 and Eclipse 3.1 development as closely as possible. We will try to synchronize with Eclipse platform development approximately every six weeks, but the schedule may change as we move forward.

Milestone Date Requirements Platform Driver
M1 January 14, 2005 Automated builds in place
Draft plan accepted by committers		 Eclipse 3.1 M4
M2 February 18, 2005 Stable build reflecting progress
Core refactor complete		 Eclipse 3.1 M4
M3 April 1, 2005 Stable build reflecting progress Eclipse 3.1 M5
M4 May 13, 2005 Stable build reflecting progress
Test plans in place		 Eclipse 3.1 M6/RC0
M5 June 10, 2005 Development freeze, feature complete Eclipse 3.1 M6/RC0
M6/RC0 June 24, 2005 Final freeze
Testing completed
Release candidate available		 Eclipse 3.1 RC1

We'll shoot for GA shortly after July 1, 2005, but this will depend on the Eclipse 3.1 schedule.
Themes and Priorities
  1. Maintainable Core Model

    2. The major theme of the Eclipse-RPM 3.0 plan is to promote lasting maintainability of the Eclipse-RPM core (represented by the org.eclipse.cdt.rpm.core plug-in). Most items in the Eclipse-RPM 3.0 plan are dependent on the core refactor.

    Proposed items:

    • 82195: Refactor org.eclipse.cdt.rpm.core (P1)
    • 82196: Move propertypage plug-in to org.eclipse.cdt.rpm.ui (P3)
    • 82197: Add RPM project nature (P3)

  2. CDT Builder Integration

    3. Proposed items:

    • 82192: Automatically add configure as external builder (P3)
    • 82194: Automatically invoke CDT builder on imported C/C++ projects (P3)

  3. Standards Adherence

    4. Proposed items:

    • 82190: Define code formatting settings for Eclipse-RPM projects (P1)
    • Fix specific violations of the Eclipse UI guidelines (bugzilla entries to follow)

  4. User Experience

    5. Proposed items:

    • 82178: Implement rpmbuild context menu actions for spec files (P3)
    • 82179: Export to a user-defined location (P3)
    • 82180: Implement text-based spec file editor (P3)
    • 82188: Integrate patch generation process with Platform compare (P3)
    • 78772: RPM Export should show patch (P3)

  5. Software Quality

    6. Proposed items:

    • 82174: Automated builds for Eclipse-RPM and Eclipse-OProfile (P1)
    • 82176: Implement unit tests (P2)
    • 82177: Update and publish manual test plans (P2)

  6. Other

    7. Proposed items:

    • 76886: Cannot export two SRPMs at the same time (P3)
    • 78773: Patch naming should guard against special characters (P3)
    • 78893: NullPointerException during RPM export (P3)
