Eclipse-RPM Developer Information

Developer Information and How to Get Involved
CVS Repository

The Eclipse-RPM sources are available under anonymous CVS access:

cvs -d :pserver:anonymous@dev.eclipse.org:/home/tools login

Press 'enter' when prompted for a password.

cvs -d :pserver:anonymous@dev.eclipse.org:/home/tools co -z9 org.eclipse.cdt-contrib/org.eclipse.cdt.rpm

Developer Communication

All communication related to Eclipse-RPM development should be directed toward the cdt-contrib-dev@eclipse.org mailing list.

Patches

We love patches! If you have a patch to submit, first open a Bugzilla ticket, then attach your patch to the open bug. A committer will review your patch and commit it to the source repository upon approval. Alternatively, you can post your patch to the cdt-patch@eclipse.org mailing list.

We use GNU-style ChangeLog entries to track changes made to source files. When you post a patch, you generally paste your ChangeLog entry above the beginning of the patch in the patch file.

Development Guidelines

Please follow the Eclipse Foundation's commonly adopted coding conventions.

In addition, when working with user interfaces, please try to adhere to the Eclipse UI Guidelines.
