OProfile Sessions

OProfile uses sessions to manage different runs of the OProfile daemon. A session is simply a named collection of sample files. OProfile uses a default unnamed session to contain all collected samples from the last run of the profiler. Every time the OProfile daemon runs, it overwrites the default session. To save the current session for later comparison, for example, you can use OProfile's opcontrol utility to save the session.

The Eclipse OProfile plug-ins use the same concept of sessions that OProfile does—with one minor difference: the OProfile default session (which has no name) is named "Default" in the user interface. It is this "Default" session that may be saved using opcontrol.
