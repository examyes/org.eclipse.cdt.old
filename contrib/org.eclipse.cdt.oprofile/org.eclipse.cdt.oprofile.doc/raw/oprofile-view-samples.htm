Viewing OProfile samples

The OProfile daemon collects events for the whole system. In this section, you will use the OProfile perspective to view data collected by OProfile.

  1. From the perspective shortcut toolbar, select the OProfile perspective (alternatively, select Window > Open Perspective > Other > OProfile).
  2. The OProfile perspective opens with the default views: System Profile view and the Sample view. These views are used to navigate and display the information collected by OProfile. In this example, OProfile has collected information about three sessions, the "default" session and two saved sessions, "session-1" and "session-2". Click on the item in the System Profile view labeled Default.

  3. The Sample view now displays information about the "default" session. It shows the equivalent of running op_time -l from the command line. Notice that the most events occurred in the executable /home/me/workspace/pi/pi.
  4. You find out more about this executable and where the samples occurred. In the System Profile view, expand the "default" session by clicking on the small triangle to the left of Default.

    A list of executables appears in the view.

  5. Select /home/me/workspace/pi/pi to view information about this executable in the Sample view.

  6. Click on the arrow beside /home/me/workspace/pi/pi to expand its contents so that you can view the list of samples or symbols in /home/me/workspace/pi/pi for which samples were collected.
  7. Select main.

    The Sample view displays information about "main". Notice that the view shows the address at which the sample was collected, the total number of samples taken at this address, the percentage of all the samples of /home/me/workspace/pi/pi that this sample represents, and any debugging information that was found for the sample.

  8. Double-click on a sample in the Sample view to open an editor at that location. In the example, the sample at address 0x080485b (/home/me/workspace/pi/pi/pi.c line 61) is shown in the editor.
