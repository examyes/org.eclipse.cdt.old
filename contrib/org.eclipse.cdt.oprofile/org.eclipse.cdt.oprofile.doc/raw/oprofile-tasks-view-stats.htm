Viewing overall system profiling statistics

The System Profile view displays sample files, that is, files that contain OProfile sample information. These sample files are displayed as the original executable for which the sample was taken. For example, the sample file /var/lib/oprofile/samples/}home}me}foo#0 would be displayed as "/home/me/foo", since this is the executable from which the event was collected.

To view sample counts for the entire system, select the OProfile event to view in the System Profile view's menu. The menu shows a list of all the events collected by all sessions in the OProfile samples directory. Note that not all events may exist in all sessions.

After choosing an event, select one of the sessions to view. The System Profile view shows all the executables for which the event was collected. (Executables can be shared libraries and kernel modules, as well as conventional executables.) The Sample view displays count information for the samples collected for the chosen session. Specifically, the Sample view shows:

  • The total sample Count for each executable for which there are samples.
  • The percentage (%age) of the total counts of this session collected in this executable.
  • The Name of the executable.

By default, the Sample view sorts the executables in order of decreasing sample count; that is, it displays the executable with the most samples at the top.
