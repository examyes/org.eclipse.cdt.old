Profiling your C/C++ application

Oprofile is a tool that enables you to profile your C/C++ application. To run Oprofile:

  1. Load the Eclipse C/C++ perspective: from the menu bar choose Window > Open Perspective > C/C++ Development.
  2. Next, from the toolbar choose Run > Run. The Launch Configurations dialog appears.
  3. In the Launch Configurations dialog, choose Profile C/C++ Application > New_configuration.

  4. Fill in each of the tabs:
    Application
    Use this tab to select a C/C++ launch configuration that the oprofile launcher will run. If you leave this field blank, oprofile gives you the profile of the "idle" system.
    Profiling Setup
    Use this tab to specify system-wide profiling options for the daemon (these settings affect all counters). The dialog queries oprofile for defaults for most of the options, except the four checkboxes at the bottom. These are designed to be reasonable defaults for most Eclipse users.

    Configuration
    Use this tab to configure the available counters.

  5. When the information is complete, click Run. The system authenticates you as root then displays data in two views:
    • A project-centric view (showing only the files in the users project)
    • A system-centric view, showing everything.
    You choose these views through the Navigator view. You can:
    • Double-click an item to see a separate display that shows all the data and counts for that item.
    • Open an editor at the places listed in the navigator.
    • Click on the "System Root" to display the data for "op_time".
    • Click on an object file (executable, shlib, module) to show "op_time -l" for that object.
    • Expand objects and get a list of symbols that have samples.

Note: The system authenticates you as root only once per login session).
