Launching OProfile

In order to collect samples for your system, you must create an OProfile launch configuration that specifies, for example, what event to collect. To create your own launch configuration, go to Run > Run and click on Profile C/C++ Application. The dialog displays the different tab groups to configure and run the OProfile daemon:

  • The Application tab enables you to specify which C/C++ launch configuration to run after the daemon is started. It is optional.
  • Click on the Browse button to browse a list of defined C/C++ launch configurations.
  • The Profiling Setup tab defines generic OProfile daemon options, such as the name of the kernel image file in use, logging verbosity, and options to separate certain types of samples. The kernel image parameter is required. The launcher will try to guess the correct image file based on the current kernel's version. If it is incorrect, enter the correct image filename or use the Browse button to browse the file system and select the correct file.
  • The Configuration tab specifies how the daemon will collect events. By default, no counters are enabled. To enable a counter, select the appropriate counter number from the combo box at the top of the tab and click the Enabled checkbox. Next, select the desired event for collection in the Counter settings area of the tab.
  • The Common tab is a standard Eclipse-defined tab that, for example, enables you to save the current launch configuration in the workbench's Run menu.
