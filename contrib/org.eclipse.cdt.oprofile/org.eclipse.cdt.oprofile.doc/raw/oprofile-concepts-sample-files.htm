OProfile Sample Files

Sample files are collections of samples that were taken by the OProfile daemon. Unless the "separate-libs" or "separate-kernel" options are used, there is only one sample file per executable that ran on the system. These sample files contain all the samples colleted in the executable image. Thus, each run of an executable causes OProfile to output only one sample file.

Using the "separate-libs" option for OProfile, you can cause OProfile to track shared library samples for each program that uses it. These show up as children of the program in the System Profile view and allow you to view a program's use of the shared library.
