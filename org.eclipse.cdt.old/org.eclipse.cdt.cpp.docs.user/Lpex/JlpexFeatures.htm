Features

General

  • written in Java (so quite portable)
  • fully programmable widget
  • HTML documentation:  reference & programming interface
  • AWT/Swing+ClassicJFace-based JavaLPEX:
    • stand-alone version
  • Eclipse SWT-based JavaLPEX plugin:
    • vertical ruler
    • marker support
    • content assist.

Editing

  • document parsers (syntax highlighting, etc.) for:
    • Java
      - including embedded SQL (*.sqlj files)
      - with Document & Insert-trace actions
    • C/C++
      - including embedded SQL and CICS (*.sqc, *.ccs files)
    • REXX
    • HTML
    • XML
    • XSL
    • COBOL
      - including support for COBOL/400 and ILE COBOL
    • HLAsm
    • PL/I
      - including embedded SQL and CICS
    • JCL
    • SQL
    • Java *.properties files.
  • filtered (zoom-out) edit views
    - filter on selected text
    - filter on syntax:
    • Java parser:  (classes+) methods (Ctrl+G), logic outline, [embedded SQL,] errors
    • C/C++ parser:  functions (Ctrl+G), logic outline, [embedded SQL,] [embedded CICS,] errors.
  • 4 block selection modes (stream, character, rectangle, element)
  • live find & find and replace
    • regular expression search
    • column-restricted search.
  • editor profiles (key and mouse behavior emulation)
    • SEU, ISPF, XEDIT - including prefix area commands
    • EPM, BRIEF, EMACS, VI.
  • marks
    - including highlighting and visibility
  • file compare
    • ignore case
    • ignore leading blanks
    • ignore trailing blanks
    • ignore all blanks
  • sort
  • printing (not yet in the Eclipse SWT-based JavaLPEX)
    - including tokenized printing
  • saving tokenized image of document as HTML file suitable for browsers
  • sequence numbers
    - with formatted display
  • text-length limit
    - sensitive to bytes in the file in its source character encoding
  • format line
  • show lines (used for embedded error messages, etc.)
  • Ctrl+M to match parentheses, braces, square brackets, and angle brackets
    - also to match language constructs in PL/I documents
  • Crtl+R to expand common templates
    - also to create basic file skeleton in empty Java and PL/I documents
    • Java parser:  if, for, while, do, switch, case, out, err, try, main, doc
    • C/C++ parser:  if, for, while, do, switch, case, try, main, doc
    • PL/I parser:  if, do, select.
  • key recording/playback
  • national languages currently available:  Simplified Chinese, Traditional Chinese, Danish, Finnish, French, German, Italian, Japanese, Korean, Norwegian, Portuguese, Spanish, Swedish.
  • DBCS support
    - including display of emulated SO/SI control characters
  • line hex editing (not yet in the Eclipse SWT-based JavaLPEX)
    - hex Unicode, hex native character encoding, optional hex host encoding
  • insert & replace (often missing in Windows native editors...) editing modes.
