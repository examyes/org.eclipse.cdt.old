Sample Project 1: studentdata

The program "studentdata" is written in C and is provided just to show the basic functionality of The CC++ Development Plugin. The source files intentionally contain a BUG in order to illustrate the different views. Follow the next steps to find it and fix it.

  1. Create a C/C++ Project
    Make sure you are in the C/C++ perspective (Perspective->Open->Other...C/C++ Development). Create a C/C++ Project (File->New->Project...C/C++ Project). Enter a project name (e.g "studentdata") and specify as its resource location the local directory that contains the source files. i.e. click on the Browse button and find the location of the folder "plugins/com.ibm.cpp.docs.user/sample/studentdata". Set the C++ Build Invocation to "make" and the Parse Include Path (optional) to the include directory of your compiler.



  2. Parsing the source files
    After the project creation, you will see the project in the Navigator view. Expand the folder by clicking on the + sign to see its contents (i.e. the source files). To parse the source files, right-click on the project (in the Navigator view) and select the Parse...Project option.



    • The System-Objects view displays the objects parsed from the include directory ( you can avoid parsing them by not specifying the "Parse Include path" from the "Properties" option). Double-click on any object and the Editor view would display the source file and you would be positioned at the line where the object occurs. Some of the parsed objects are themselves expandable (e.g. functions) so you can click on the + sign to see the content of the functions.
    • The Parsed-Files view contains the list of source files that have been parsed.
    • The Details view displays the contents of the object selected in the Parsed-Files view

    NOTE: If you want to specify the types of files to parse such as .cc .C etc. you can add them to the "<homedir>/eclipse/plugins/com.ibm.dstore.core/CPPExtensions.dat" file.

  3. Build (Compiling) projects
    By default the Automatic Build Feature is enabled so the project builds automatically once a resource (e.g. a source file) has been modified. Our project was built automatically upon creation but you can disable this option (->Window->Preferences).
    If you disable Automatic build, you will need to right-click on the project and select the Rebuild All option (A build will execute whatever command was specified as the Build invocation, in our case it would be make). Any output that the build produces (e.g. error messages or warnings) will be displayed in the Output view.



  4. Handling build errors
    The sample code provided contains a BUG and therefore errors are displayed in the Output view upon a Build. Clicking on any of these errors will redirect you to the place in the source file where the error was detected. e.g click on the first reported error and you will be taken to the place where it occurs.

    Since we meant TRUE instead of TRUUE, change TRUUE to TRUE. Building the project should report no errors now.

  5. RUNNING the project
    You run the newly created executable from the Command view. e.g. for the studentdata project enter "studentdata < input.test" as the command to run in the Command view. The output of the program will be displayed in the Output view.



  6. Closing the project
    Go to the Navigator view, right-click on the project to open its submenu, and select the Close option.
