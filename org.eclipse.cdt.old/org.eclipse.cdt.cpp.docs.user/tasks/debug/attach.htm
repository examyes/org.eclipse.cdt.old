Attaching to C/C++ Application
Steps to Attach to a C/C++ Application

To create a debug launch configuration that will attach to a compiled language application, do the following:

  1. Select the executable from a C/C++ Project and do one of the following:
    • Select Run > Debug from the workbench menu bar to open the Launch Configurations dialog box.
    • Select the workbench toolbar Debug push button and choose Debug from the menu.
    This will open the Launch Configurations dialog box.
  2. In the Launch Configurations dialog box, do one of the following:
    • Select the C/C++ Debug Attach to Application node and click New.
    • Right-click the C/C++ Debug Attach to Application node and select New from the pop-up menu.
    This will cause the launch configuration tabs and entry fields to display on the right-hand side of the dialog box.
  3. In the Name field, enter a name of your choice for the configuration that you are creating.
  4. Select the Attach to tab to bring it to the foreground.
  5. The name of the selected executable can be found in the Attach to tab Program field. You may change this field by clicking the Browse button and selecting another executable to attach to.
  6. In the Attach to tab Program process ID field, enter the process id of the program to attach to. You may click the Browse button to bring up the Select process id dialog. Select the desired process to attach and click OK. Completion of this field is mandatory.
  7. Select the Common tab to bring it to the foreground.
  8. If you want the launch configuration to be stored locally and available for a single user, ensure that the Local radio button is selected. If the launch configuration is to be stored as a file in the workspace and shared in a repository for team use, select the Shared radio button and enter the project location where you wish the launch configuration to be stored in the Location of shared configuration field. Alternatively, you may browse for the project.
  9. In the Switch to/Open perspective group, choose the perspective that you would like to have opened after the launch. This selection is made in the Debug mode field drop-down box.
  10. If you would like the launch configuration to appear as a favorite debug configuration in the Debug push button favorites menu or the Debug menu Debug History list, select the Debug check box.
  11. If all information mandatory to the launch configuration has been provided, the Debug push button will be enabled. Clicking Debug without first clicking Apply will launch a debug session from the new launch configuration and save the new launch configuration.

    Note: When you save a launch configuration for attaching to a running process, the process ID that you enter is saved. You must edit the configuration if you wish to attach to another process.

    Tip: If you have made entries in the launch configuration settings that you would like to remove or change, clicking Revert will remove all changes that you have made.
  12. If information mandatory to the launch configuration has not been provided or if there are errors in the information that you have provided, messages at the top of the dialog box will indicate what is missing. If you choose to ignore these messages, clicking Apply will save the launch configuration and you can then click Close to exit the dialog box.
  13. After you have saved a debug launch configuration, you may edit it. For further information about editing debug launch configurations, see the related task below.
Note: Unexpected results may occur if an incorrect process ID is entered or if you try to attach to a system ID.
