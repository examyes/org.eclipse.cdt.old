Debugging a C/C++ Application
Steps to Debug a C/C++ Application

To create a launch configuration that will load a C/C++ application for debugging, do the following:

  1. From a C/C++ project, select the executable you want to debug and do one of the following:
    • Select Run > Debug from the workbench menu bar to open the Launch Configurations dialog box.
    • Select the workbench toolbar Debug push button and choose Debug from the menu.
    This will open the Launch Configurations dialog box.
  2. In the Launch Configurations dialog box, do one of the following:
    • Select the C/C++ Debug Application node and click New.
    • Right-click the C/C++ Debug Application node and select New from the pop-up menu.
    This will cause the launch configuration tabs and entry fields to display on the right-hand side of the dialog box.
  3. In the Name field, enter a name of your choice for the configuration that you are creating.
  4. Select the Debug tab to bring it to the foreground.
  5. You will find the name of the selected executable entered in the Debug tab Program field. You can change this field if you desired to run another executable from the project.
  6. In the Debug tab Program parameters field, enter program parameters to be passed to the debugee.
  7. In the Debug tab Working Directory field, enter the directory you wish to switch to before the debug session is launched. This field is defaulted to the directory in which the selected executable resides. Click the Browse button to choose a different directory.
  8. Select the Common tab to bring it to the foreground.
  9. If you want the launch configuration to be stored locally and available for a single user, ensure that the Local radio button is selected. If the launch configuration is to be stored as a file in the workspace and shared in a repository for team use, select the Shared radio button and enter the project location where you wish the launch configuration to be stored in the Location of shared configuration field. Alternatively, you can browse for the project.
  10. In the Switch to/Open perspective group, choose the perspective that you would like to have opened after the launch. This selection is made in the Debug mode field drop-down box.
  11. If you would like the launch configuration to appear as a favorite debug configuration in the Debug push button favorites menu or the Debug menu Debug History list, select the Debug check box.
  12. If all information mandatory to the launch configuration has been provided, the Debug push button will be enabled. Clicking Debug without first clicking Apply will launch a debug session from the new launch configuration and save the new launch configuration.

    Tip: If you have made entries in the launch configuration settings that you would like to remove or change, clicking Revert will remove all changes that you have made.
  13. If information mandatory to the launch configuration has not been provided or if there are errors in the information that you have provided, messages at the top of the dialog box will indicate what is missing. If you choose to ignore these messages, clicking Apply will save the launch configuration and you can then click Close to exit the dialog box.
  14. After you have saved a debug launch configuration, you may edit it.
