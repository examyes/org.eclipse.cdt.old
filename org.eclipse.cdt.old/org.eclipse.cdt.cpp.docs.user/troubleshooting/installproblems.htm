Troubleshooting Installation/Invocation Problems

If you encounter problems with your installation of the C/C++ Perspective or if after a period of time the Perspective stops working, first ensure the following files exist:

  • <...>\eclipse\plugins\com.ibm.cpp.core\cpp_core.jar
  • <...>\eclipse\plugins\com.ibm.cpp.core\plugin.xml
  • <...>\eclipse\plugins\com.ibm.cpp.core\runCommand.bat
  • <...>\eclipse\plugins\com.ibm.cpp.core\formats.dat
  • <...>\eclipse\plugins\com.ibm.cpp.ui\plugin.xml
  • <...>\eclipse\plugins\com.ibm.cpp.ui\cpp_ui.jar
  • <...>\eclipse\plugins\com.ibm.lpex\vabaselpex.jar
  • <...>\eclipse\plugins\com.ibm.lpex\plugin.xml

If any one of these files is missing, you will need to re-install the C/C++ Development Plugin.

In some circumstances we've noticed that the workbench metadata becomes corrupt. If this happens, you usually receive an error dialog about an "unrecoverable error in the desktop". Our workaround has been to delete the <...>\eclipse\workspace\.metadata directory. If you have a lot of information in the workbench that you do not want to lose, then prior to wiping the directory, save each project sub-directory to a temporary location, then you can re-import your source after deleting the workbook.

If all of the above files exist and your workbench is not corrupt, and you are still having problems bringing up the C/C++ Perspective, then you have either run into a bug/limitation in our plugin, or perhaps a bug in the Studio Workbench itself. Please let us know of such situations with a complete description of your problem.
