Description

The CDT (C/C++ Development Tools) Project is working towards providing a fully functional C and C++ Integrated Development Environment (IDE) for the Eclipse platform. Our focus is development on Linux for deployment on Linux, but we are interested in participation from others who would like to extend our work in other directions (e.g. a Windows client, targeting Unix(R) or embedded platforms, wizards for developing applications that use particular library or database or messaging APIs, or extension to other languages). We are looking for contributions from the open source community in the areas of testers, developers and general users who can help us ensure that the C/C++ tools work well on all the Eclipse platforms.

Our first release function includes:

  • Support for the "autoconf" and "automake" utilities to define new projects or in existing projects imported into the environment. However, use of these tools is not mandatory, and your build may consist of hand-generated makefiles, Perl scripts, or whatever, as long as it can be launched from a command line.
  • Easily configurable error message parser, preconfigured for gcc and g++.
  • Code browsing, navigation, automatic code completion provided by an integrated C/C++ parser and program database.
  • Integrated debugger GUI using gdb, reusing many of the familiar Eclipse debug views but adding new ones for registers, storage, and a debugger console that provides access to the underlying gdb command line interface for advanced or esoteric debugger function.
  • Integrated access to the platform man, info, and HTML help. Select a keyword in the editor and hit F1 to search, results are displayed using the Gnome help browser or Konqueror according to your preference.
  • Support for projects that are remote, on another Linux system, as well as local development on your Linux desktop or laptop. Projects can be replicated between systems, so you can work locally for a while then replicate to a second Linux system where you rebuild and remotely edit and debug from within the IDE. The mechanism supporting this is extensible and only requires a Java 1.3 runtime on the remote Linux system, in addition to any underlying system utilities you may choose to invoke (gmake, gcc, gdb, perl, etc).

Status

Our work is now at what we describe as a 0.5 level, meaning it isn't yet ready for production use but has progressed to the point where we feel comfortable providing drivers for interested developers or users to experiment with. Inconsistent or incomplete function, scalability, and stability problems may occur.

Platforms

The plugins are written in Java with no native code and thus may be ported to any platform supported by Eclipse. However, some Linux specific features prereq other tools that may not be found on non-Linux systems.

Content

We provide a total of 15 plugins which are grouped into 7 components which each behave as an autonomous project:

  1. Core
    • org.eclipse.cdt.dstore.core - base Data Store metadata database implementation
    • org.eclipse.cdt.dstore.extra - client-side Data Store extensions
    • org.eclipse.cdt.dstore.extra.server - server-side Data Store extensions
  2. Generic-Extensions
    • org.eclipse.cdt.dstore.miners - remotable actors for generic command invocation
  3. UI
    • org.eclipse.cdt.cpp.ui - contributions of a C++ Perspective, views,and actions
    • org.eclipse.cdt.dstore.ui - ui Data Store related general views and actions
    • org.eclipse.cdt.dstore.hosts - remote system support
  4. Debug
    • com.ibm.debug - client-side extensions for debugging non-Java applications
    • org.eclipse.cdt.debug.gdbPicl - server-side wrapper for GDB
  5. Cpp-Extensions
    • org.eclipse.cdt.cpp.miners - remotable actors for build, launch, and project management
    • org.eclipse.cdt.cpp.miners.parser - the remotable C/C++ source parser
  6. Doc
    • org.eclipse.cdt.cpp.docs.user - online Eclipse documentation
    • org.eclipse.cdt.linux.help - provides access to Linux platform-supplied help and reference information
    • com.ibm.debug.doc.user - Eclipse documentation for the above
  7. Lpex-Editor
    • com.ibm.lpex - a Java implementation of the IBM Live Parsing Editor
