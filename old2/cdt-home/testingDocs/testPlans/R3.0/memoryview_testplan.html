1. Introduction
1.1 Document Purpose

This test plan defines the test strategy and test cases for testing the feature under test.  The feature under test is Debug Memory View,  specifically QNX commitments for CDT 3.0.

The test plan ensures that the feature's implementation satisfies the stated requirements stated in the SRS, and that the feature's design elements are of high quality.  This test plan may also test project-level requirements (scalability, internationalization, etc.) as they relate to this feature.

1.2 References

The feature is outlined in Memory View feature document. 

The delivery schedule of this feature is specified in CDT3.0 Master Project Plan

The configurations, reporting criteria, general risks, etc. for the CDT 3.0 project are specified in the Master Test Plan.

3. Test Cases 
Test ID Synopsis Steps
T1.1 basic workflows  open Memory view from Windows/Show Views/Memory, close the view, verify the no surprises.
T2.1 Adding memory monitor from Memory monitors pane

  1. Start a debug session.

  2.  When session is suspended select “Add Monitor” in the context menu of the Memory Monitor pane. Verify that the menu is disabled if session is not suspended.

  3.  Type an address or expression in the “Enter address or expression field” field and press OK. Verify that memory monitor is added and default rendering tabs are created in the Memory Renderings pane.

  4. Verify the base address is highlighted in memory renderings pane
T2.2 Adding memory monitor from Memory Renderings  pane

  1.  Start a debug session.

  2.  When session is suspended select “Add Rendering” in the context menu of the Memory Renderings pane.

  3.  Press the “Add New…” button.

  4. Type an address or expression in the “Enter address or expression field” field and press OK.

  5. Select renderings from the list and press OK. Verify that memory monitor is added and rendering tabs are created in the Memory Renderings pane.

  6. Verify the base address is highlighted in memory renderings pane
T2.3 Adding memory monitors by using  different expressions repeat T2.1 and T2.2 but using different expressions: hexadecimal, variable's name and pointers.
T3.1 Removing Memory monitors
  1. Select a memory monitor, Remove a memory monitor by using "Remove Memory Monitor" from context menu or the button
  2. Verify the corresponding memory monitor is removed
T3.2 Remove all  Memory monitors
  1. Remove all  memory monitor by using "Remove Memory Monitor" from context menu or the button
  2. Or, Select multiple memory monitors and use "Remove Memory Monitor" from context menu to remove all of them
  3. Verify all of memory monitors are removed
T4.1 Add a new memory rendering
  1. Select an memory monitor from memory monitors pane, on a memory renderings pane, select "Add Rendering” in the context menu of the Memory Renderings pane.
  2. Verify the selected memory monitor is the one displayed in Memory Monitors dropdown list, choose another if you want
  3. select one from memory rendering(s) type list, then press "OK"
  4. Verify a new rendering is created with memory monitor's name, value and rendering type display on the tab
  5. Repeat above steps but choose different types
T5.1 Removing memory rendering
  1.  On a memory renderings pane, select "Remove Rendering” in the context menu of the Memory Renderings pane.
  2. Alternatively, press the "X" button on right corner of Memory renderings pane
  3. Verify the memory rendering is removed
T6.1  Use Memory view options: Toggle Memory monitors pane Verify the memory monitors pane can be toggled on and off by pressing the "Toggle Memory Monitors Pane" button on top right corner of memory view
T6.2 Use Memory view options: Toggle split pane  Verify the memory rendering pane can be splitter by pressing the "Toggle split pane " button on top right corner of memory view
T6.3 Use Memory view options: Link Memory Rendering Panes
  1. Press down  the button of "Link Memory Rendering Panes"
  2. Create 2 renderings and make them splitted
  3. Verify the selection of memory content from one rendering is synchronized with another
T6.4 Use Memory view options: Padded String Verify the string that represents the un-retrievable memory can be specified with "Padded String" option
T6.5 Use Memory view options: Default Column Size
  1. Verify the default column size of the memory presentation (1 , 2 , 4 , 8 or 16 bytes) can be specified. 
  2. Verify the default setting can be overwritten by each rendering.
T6.6 Change Color and font of memory view
  1. From Windows/Preference/General/color and fonts, change the font for displays in memory rendering
  2. Verify the fond changed accordingly
  3. From Windows/Preference/Run and Debug, change the color of buffered and un-buffered lines in memory renderings pane
T7.1 Memory rendering operation: scrolling ability Scroll the content of a memory rendering up and down.
T7.2 Memory rendering operation: context menu-1 Try “Go to address” action.
T7.3 Memory rendering operation: context menu-2 Try “Restore base address” action.
T7.4 Memory rendering operation: context menu-3 Try other menu options
T8.1 Memory changes during debugging Monitor the memory changes occurred during debugging(stepping).
T9.1 Modify Memory content

Modify the content of overlapping memory monitors.

T9.2 Modify Memory content - overlapping

Modify the content of overlapping memory monitors.
T9.3 Variable view changes after Memory content modified Check if memory modifications are reflected in the Variables view.

  1. Create a memory monitor based on a variable.

  2.  Modify the content of variable in the memory view. Verify that the value of the variable is updated.

  3.  Modify the variable’s value. Verify that changes are reflected in the memory view.

T10.1 Memory view for multiple debug sessions Test the behavior of the memory monitors created for multiple debug sessions. Check adding and removing monitors/renderings and  modifying contents...
T10.1 Memory view for multiple debug sessions - scalability Test the behavior of the memory monitors created for multiple debug sessions(5 or 6).  
T11.1 Error handling - invalid memory monitor expression Enter an invalid expression in the “Add memory monitor” dialog. Verify that the error is reported
T11.2 Error handling - Invalid value of memory content Click a memory content, and enter an invalid value, such as "kkk" for hex, or "abc" for integer, verify the error dialog pop up 
T12.1 Multiple entries in memory monitors pane Keep adding memory monitors and make sure no exceptions or errors occurred.
T12.2 Multiple entries in memory renderings pane Keep adding memory renderings and make sure no exceptions or errors occurred.
T12.3 Static lib Check how the memory view handles static lib. 
T12.4 shared  lib Check how the memory view handles shared  lib. 
T12.5 DLL loaded with  dlopen() Check how the memory view handles dll  lib.
