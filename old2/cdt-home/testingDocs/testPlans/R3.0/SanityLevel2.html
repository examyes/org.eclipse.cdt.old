Notes

Before you begin

  1. If you have a copy of LogAlert to monitor your <workspace>/.metadata/.log file, start it now. (Windows only)
  2. Disable the auto-build feature before you begin.
  3. For Windows,  please install eclipse and CDT onto "Program Files".  This will add a space in the path of the workspace and all related files of the CDT.
  4. If you are running eclipse with the IBM JRE, create a short cut of eclipse.exe and in the properties, make sure you should have ".../eclipse.exe" -vmargs -Xj9.  Verify that in Help->About Eclipse Platform->Configurations Details", you are running with the J9 option.

Create a project
  1. Right Click in the Navigator view and select New -> Project or select File -> New -> Project from the toolbar.
  2. >From the new Project wizard select C++ -> Managed Make C++ Project.
  3. Enter MgdCpp in the Project name field and click next if you want to change the default build target or hit Finish.
  4. Verify that the project is added to the workspace and it is opened in the C/C++ Perspective.

Delete a project
  1. Select MgdCpp in the C/C++ Perspective. Right Click in the project and select Delete.
  2. Make sure that the Confirm Project Delete dialog comes up. Select 'Do not delete contents' radio button. This deletes the project from the C/C++ perspective but it is still there in the workspace.
  3. Verify that the project and its contents are deleted from the C/C++ Perspective but still exist in the workspace.

Import a project

Note: Bring up the Progress view and make sure it is visible while importing the projects. (how to bring up the Progress view)

  1. Right Click in the C/C++ Perspective and select Import or select File -> Import from the toolbar.
  2. >From the Import dialog, select Existing Project from Workspace.
  3. Click on the Browse button for the Project contents field and look for MgdCpp in the workspace. Select MgdCpp and click Finish.
  4. Verify that the project has been imported along with its contents and is open in the C/C++ Perspective.
  5. Make sure that the Indexer is kicked off after the project is imported.

Close a project
  1. Ensure there is an executable in the binaries folder under the MgdCpp project.
  2. Right Click on MgdCpp in the C/C++ Perspective and select Close Project.
  3. Verify that the project is visible in the C/C++ Perspective but there are no contents in it.

  3. Make sure that the Indexer is kicked off after the project is opened.

  1. Create a managed C++ project with main.cpp containing the code, where 30 seconds is the amount of time you need to kill a process. If using RedHat linux you may not be able to call sleep, so use nested loops of 1000.

    2. int main(int argc, char **argv) {

      sleep(30);

  3. Open Signals view and verify SIGINT has Suspend=yes, Pass=no.

    kill -s INT `ps -ef | grep gdb | head -n1 | awk '{print $2}'`
  6. Verify program suspends with the Debug view showing Signal 'SIGINT' received.
  7. Run to end.
  8. Start another debug session stopping in main.
  9. Set SIGINT: Pass=No, Suspend=No.
  10. Continue.
  11. While sleeping send signal SIGINT.
  12. Verify the program ignores the signal and continues to end.
  13. Start another debug session stopping in main.
  14. Set SIGINT: Suspend=Yes.
  15. Continue.
  16. While sleeping send signal SIGINT.
  17. When the program is suspended, set SIGINT: Pass=yes.
  18. Continue.
  19. Verify program terminates without printing numbers.

  20. Start another debug session stopping in main. Continue.
  21. Right click SIGINT and select Resume with signal.
  22. Verify program terminates.

Scanner config

Test 1
  1. Unzip SC.zip into a local directory.
  2. Import the project using the Import Existing Project into Workspace wizard. Make sure the project properties/binary parser is set correctly (Windows PE parser for windows, Elf parser for linux).
  3. Build the project. It may not compile yet, but we need the build action so the scanner is invoked.
  4. Open Properties > C/C++ Project Paths > Path Containers > Discovered Paths. You should see the include paths and defined symbols that the scanner configuration feature discovered. The format of the paths will depend on the platform where you are running the test, but the list should contain a variation of :
    • /usr/include/c++/<version>
    • /usr/include/c++/<version>/<platform>
    • /usr/include/c++/<version>/backward
    • /lib/gcc-lib/<platform>/<version>/include
    • /usr/include
    where version is the compiler version and platform identifies the processor and OS type (i.e. i686-pc-cygwin). The defined symbol list will contain different values depending on the OS the test is running on. Typically, you will see symbols for the version, minor, and patch level, along with symbols identifying the OS.
    5.
  5. Now let’s add a new path and make sure it gets discovered. Create a file. Call it findme.h and copy-paste the code by following the hyperlink. Copy this file to C:/include on your file system. Do not save this file in the workspace. Note that if you are testing on a POSIX system, change C:/include to a directory that you have write privileges in.
    6.
  6. Modify the makefile in the test directory of the project so that INC_PATHS points to the correct location of findme.h.
    7.  
    SRCS += test/Test.cpp
    INC_PATHS := /directory/containing/findme.h/with/write/permissions
     

    test/%.o: test/%.cpp

           $(CC) $(CCFLAGS) -I$(INC_PATHS) -o $@ $<
  7. Build the project.
  8. Search for the declaration of FOUND_ME. To do that, open test/Test.cpp, click on the reference and hit F3. You should be taken to the header file where it is declared.
  9. Open Properties > C/C++ Include Paths & Symbols. Expand the project and note that discovered path list should now contain the directory C:\include (or your path with write permissions). The problem is that we do not want this path applied to whole project, just source files built in the test folder. Select Discovered Paths > Edit … > C:\include > Disable > Finish to make sure it does not keep being added back to the list.
  10. Now we are going to manually add it back so it only applies to the test folder and retest that search still works. Select Add Folder/File > test to add the folder to the list control. Select test > Add External Include Path… > C:\include > OK.
  11. Rebuild the project. Search for the declaration of FOUND_ME and ensure that you are still getting the external header file as the result.
