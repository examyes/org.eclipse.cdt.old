Notes

Before you begin

  1. If you have a copy of LogAlert to monitor your <workspace>/.metadata/.log file, start it now. (Windows only)
  2. Disable the auto-build feature before you begin.
  3. For Windows,  please install eclipse and CDT onto "Program Files".  This will add a space in the path of the workspace and all related files of the CDT.
  4. If you are running eclipse with the IBM JRE, create a short cut of eclipse.exe and in the properties, make sure you should have ".../eclipse.exe" -vmargs -Xj9.  Verify that in Help->About Eclipse Platform->Configurations Details", you are running with the J9 option.

Create Project

Steps

  1. Create a new managed make C++ project, call it MgdCpp. (how to create a project)
  2. Create a new standard make C++ project, call it StdCpp.
  3. Create a new managed make C project, call it MgdC.
  4. Create a new standard make C project, call it StdC.
  5. Verify that all four projects were added to the workspace and that they were opened in the C/C++ Perspective.

Import Existing C++ Project into Managed C++ Project

Steps

    Note: Bring up the Progress view and make sure it is visible while importing the projects. (how to bring up the Progress view)
  1. >From the OS file manager, save shape.zip on your system.
  2. >From Eclipse, right click on MgdCpp and click Import. Select "Zip File". Browse for shape.zip and import it to MgdCpp.
  3. Verify that the project has been imported and is open in the C/C++ Perspective.
  4. Make sure that the Indexer is kicked off after the project is imported.
    5.  
Build Managed Make Project

Steps

  1. Select MgdCpp and initiate a build.
  2. Verify that a build was executed successfully.

Run Managed Make Project

Steps  

  1. Select MgdCpp in C/C++ Projects view.
  2. Create a new Run configuration (how to create a Run configuration)
  3. Run the configuration.
  4. Verify that execution of the program produces no errors.

 

Import Existing C++ Projects into the workspace

Steps

  1. Choose Window->Close All Perspectives.
  2. Open the "Resource" Perspective.
  3. Turn off "Build Automatically".
  4. >From the OS file manager, save 3Projects4sanity.zip on your system.
  5. Unzip the zip file into your workspace. (Note: Always do a fresh unzip before proceeding to the following steps)
  6. Display the Progress view and make sure it is visible while importing the projects. (how to bring up the Progress view)
  7. >From Eclipse, File->Import...
  8. Choose "Existing Project into Workspace"
  9. Browse to "MessageLibrary" and "Finish".
  10. Make sure that the Indexer is kicked off after the project got imported.
  11. Verify that the project has been imported and is open in the C/C++ Perspective.
  12. Redisplay Progress View because it was not part of C/C++ perspective.
  13. Repeat 7 to 12 with "UIComponent" and "Executable" projects.
Build Executable Project

Steps

  1. Make sure that the Progress View and the Console View are both visible simultaneously. (how to bring up the Progress view)
  2. Expand UIComponent in C/C++ Projects view and copy the file cygmsgs.dll under it to Executable/Debug.
  3. Right-click on the Executable project in the C/C++ Projects view and select "Build Project".
  4. Select "Run in background".
  5. Verify that a build was executed successfully.
Debug Managed Make Project (Executable)

Steps   

  1. Right-click on the Shell project and select "Properties"
  2. Go to C/C++ Make Project->Binary Parser, select "PE Windows Parser" and click OK.
  3. Open main.cpp under the Executable project.
  4. Set a break-point on the second line, that is, runner.runtest(), in main().
  5. Create a new Debug configuration. (how to create a new Debug configuration)
  6. Under the 'Debugger' tab, check the 'Stop at main()' checkbox.
  7. Run the configuration.
  8. Verify that the program paused at main().
  9. Run to the break-point (F8 is the shortcut for this step).
  10. Verify that execution is paused at the second line in main().
  11. Resume.
  12. Verify that execution has ended without any errors. It will display a funny message at the end though but there should be no error.
Debug Standard Make Project (StdCpp)

Steps   

  1. Right-click on the StdCpp project and select "Properties"
  2. Go to C/C++ Make Project->Binary Parser, select "PE Windows Parser" and click OK.
  3. Create a file  main.cpp under the StdCpp project. Type the following in main.cpp:
#include <iostream>

 

using namespace std;

 

int main (int argc, char **argv) {

        cout << "Debug Test for Standard project\n";

        cout  << "Starting test case!\n";      // **(A) ** 

        for (int i=0; i<10; i++) {

            cout  << "Printing number " << i << "\n";

        }

        cout  << "Ending test case!\n";       // ** (B) **

}

  1. Set a break-point on the line A and line B.
  2. Create a file under StdCpp. Call it makefile. Follow the link makefile and copy the text from there into the makefile under StdCpp.
  3. Save the files and initiate a build for StdCpp.
  4. Verify that the build produces no errors.
  5. Create a new Debug configuration. (how to create a new Debug configuration)
  6. Under the 'Debugger' tab, check the 'Stop at main()' checkbox.
  7. Run the configuration.
  8. Verify that the program paused at the first statement in main().
  9. Run to the first break-point (F8 is the shortcut for this step).
  10. Verify that execution is paused at line A in main().
  11. Resume.
  12. Verify that the execution is paused at line B in main().
  13. Resume.
  14. Verify that execution has ended without any errors.
