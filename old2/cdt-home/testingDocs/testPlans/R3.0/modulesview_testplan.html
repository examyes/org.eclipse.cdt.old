1. Introduction
1.1 Document Purpose

This test plan defines the test strategy and test cases for testing the feature under test.  The feature under test is Debug Modules View,  specifically QNX commitments for CDT 3.0.

The Modules view of the Debug perspective lets you view information about the modules loaded in the current debug session, including executables and shared libraries. The view consists of two areas: the modules tree and the detail pane. The detail pane displays the detail information for the module selected in the modules tree. Expanding a module users can view the module’s internals - functions, global variables, associated source files, etc. Two context menu options are provided:

Load Symbols : Loads the symbols of the module. The symbols will be loaded from the file specified in the module’s properties.

Load Symbols For All: Loads the symbols of the all modules involved in the current session. This option does not affect the modules whose symbols are already loaded.

The test plan ensures that the feature's implementation satisfies the stated requirements stated in the SRS, and that the feature's design elements are of high quality.  This test plan may also test project-level requirements (scalability, internationalization, etc.) as they relate to this feature.

3. Test Cases 
Test ID Synopsis Steps
T1.1 Basic workflows
  1. Use  attached main.c and display.c to build a project and launch a debug session
  2. Open modules view 
  3. Verify all the modules are displayed (including executables)
  4. Select a module from modules tree, verify more information displayed in the detail pane
  5. Expanding a module, verify the module's internal structure is displayed: functions. global variables, etc..
T2.1 Display modules for a shared lib project Check if all modules are shown in the view. Needs a program that loads and unloads a DLL during the execution (may use  attached main.c and display.c).

  1. Start a debug session.

  2. When the session is suspended switch to the Modules view. Verify that all modules are shown correctly, including the detail information for each module.

  3.  Step through the code until the DLL is loaded. Verify that DLL is added to the view.

  4. Step through the code until the DLL is unloaded.

  5. Verify that DLL is removed from the view.
T3.1 Loading Symbols (Linux, QNX) Check if the symbols related operations. Needs a program that loads a DLL during the execution. (may use  attached main.c and display.c).

  1. Start a debug session. Make sure that the “Load shared libraries symbols automatically” option of the Shared Libraries tab is checked (default).

  2. When the session is suspended switch to the Modules view. Verify that symbols are loaded for all modules that are compiled with the debug information.

  3. Uncheck the “Load shared libraries symbols automatically” option and start a new session.

  4. When the session is suspended switch to the Modules view. Verify that symbols are not loaded for all modules.

  5.  Select a module and load it’s symbols by using “Load Symbols” from the module context menu. Verify that symbols are loaded for this module.

  6.   Select “Load Symbols for All” from the view’s context menu or press the button on modules view. Verify that symbols are loaded for all modules that are compiled with the debug information.

  7.  Step through the code until the DLL is loaded. Verify that DLL’s symbols are not loaded.

  8.   Select the DLL and load it’s symbols by using “Load Symbols” from the context menu. Verify that symbols are loaded for the DLL.

T4.1 Working with multiple debug sessions Launch multiple debug sessions and repeat T2.1 and T3.1, make sure no surprises occurred

#include <stdlib.h>
#include <stdio.h>
#include <dlfcn.h>

#define VALUE 5

int calling_lib (int z){
	void *handle;
	int (*display)(int);
	int y;
	const char *error;
	
	printf("About to call display()\n");
	handle = dlopen ( "/tmp/libdisplay_g.so",RTLD_LAZY);
	if (!handle){
		fputs (dlerror(), stderr);
		exit(1);
	}
	display = dlsym(handle, "display");
	if ((error = dlerror()) != NULL)  {
		fputs(error, stderr);
		exit(1);
	}
	y = (*display)(z); // <- regular breakpoint
	dlclose(handle);
	return (y);
}

int main(int argc, char *argv[]) {
	int y, x, z;
	
	printf("Testing shared lib dll\n");
	for (x = 0; x < VALUE; x++){
    	z = x + 1; // <-conditional breakpoint x==2
    } 
	y = calling_lib(z); // <- regular breakpoint
	printf("The y is %d and z is %d\n",y,z);
	return (0);
}
