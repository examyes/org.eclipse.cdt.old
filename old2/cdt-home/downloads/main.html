Latest Downloads 


As of CDT 1.1.0 GA, builds are now available through the Update Manager of Eclipse. To access the Update Manager, select Software Updates->Update Manager from the Help menu. To access an update site, select New->Site Bookmark from the context menu in the Feature Updates view. In the New Site Bookmark dialog, enter a name for the site (any name) and enter the appropriate URL for the update site. You can then expand the site bookmark in the Feature Updates view to reveal the available downloads.

The release, or GA, builds for the CDT are available from the CDT Release update site at the following URL: http://update.eclipse.org/tools/cdt/updates/release

The nightly builds for CDT 1.2 are available from the CDT 1.2 Builds update site at the following URL: http://update.eclipse.org/tools/cdt/updates/builds/1.2

The nightly builds represent works in progress for the upcoming releases and are provided “as is”. It is possible that these builds have major bugs and may not run at all. Always check the test logs available from the site home page (click on the link above or click on the Site home page link in the Update Manager) before installing a nightly build.

Freeze builds (F#) are considered relatively stable, no new features will be added during this development cycle. Once a freeze build has been used for a few days it will be considered a stable build. If problems are found in a Freeze build, please submit a bug report.

The CDT Project (like the Eclipse Project) uses bugzilla as its bug and feature tracking system. Entering a bug\feature report is as simple as filling in a web form on the eclipse bugzilla page. The first time you enter a bug you will need to create a new bugzilla account for yourself by providing an email address and choosing a password.

If you prefer not to use the Update Manager, the old style zip distributions are still available for GA builds.
