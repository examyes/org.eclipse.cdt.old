Please note:
Until further notice, Nightly builds will require that you remove any previous CDT plugins and/or CDT feature files from your Eclipse environment before unzipping the downloaded zip file into the Eclipse install directory.
CDT Nightly builds have not been tested beyond a simple sanity test and are provided "as is". It is possible that these builds have major bugs and may not run at all.
Freeze builds (F#) are considered relatively stable, no new features will be added during this development cycle. Once a freeze build has been used for a few days it will beconsidered a stable build. If problems are found in a Freeze build, please submit a bug report.

The CDT Project (like the Eclipse Project) uses bugzilla as its bug and feature tracking system. Entering a bug\feature report is as simple as filling in a web form on the eclipse bugzilla page. The first time you enter a bug you will need to create a new bugzilla account for yourself by providing an email address and choosing a password.
