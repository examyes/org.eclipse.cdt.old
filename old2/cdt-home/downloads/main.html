CDT 1.2 Now Generally Available

(Nov 25/03 - CDT 2.0 M5 Release Candidate 1 available)

November 25, 2003

The CDT 1.2 and CDT 1.1 builds are now available through the Update Manager of Eclipse. To access the Update Manager, select Software Updates->Update Manager from the Help menu. To access an update site, select New->Site Bookmark from the context menu in the Feature Updates view. In the New Site Bookmark dialog, enter a name for the site (any name) and enter the appropriate URL for the update site. You can then expand the site bookmark in the Feature Updates view to reveal the available downloads.

CDT 1.2 Note: For CDT 1.2, the standard make project type and the new managed make project type have been separated out from the CDT "Platform".  When using the update site, please install the CDT feature as well as the Standard Build feature and optionally the Managed Make feature, in that order, or optionally selecting them and installing them at the same time (Process All). Other release notes for CDT 1.2 are available here.

The release, or GA, builds for the CDT are available from the CDT Release update site at the following URL: http://update.eclipse.org/tools/cdt/updates/release

The nightly builds (or more accurately "regular" builds) for CDT 2.0 will available soon. The nightly builds represent works in progress for the upcoming releases and are provided “as is”. It is possible that these builds have major bugs and may not run at all. Always check the test logs available from the site home page (click on the link above or click on the Site home page link in the Update Manager) before installing a nightly build.

The CDT Project (like the Eclipse Project) uses bugzilla as its bug and feature tracking system. Entering a bug\feature report is as simple as filling in a web form on the eclipse bugzilla page. The first time you enter a bug you will need to create a new bugzilla account for yourself by providing an email address and choosing a password.
