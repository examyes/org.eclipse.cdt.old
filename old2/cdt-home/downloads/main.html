Release Builds Update Site

CDT release builds are made available on the CDT release update site. Use the following URL in a Site Bookmark in the update manager:

http://update.eclipse.org/tools/cdt/updates/release

To access the Update Manager, select Software Updates->Update Manager from the Help menu. To access an update site, select New->Site Bookmark from the context menu in the Feature Updates view. In the New Site Bookmark dialog, enter a name for the site (any name) and enter the URL for the update site. You can then expand the site bookmark in the Feature Updates view to reveal the available downloads.

Currently, CDT releases 1.2 and 1.1 are available as well as releases of CDT/CPPUnit Integration. As new releases are made and new features are added, they will appear in this update site.

For those who need the flexibility of the old zip distributions, they are available here.

CDT 1.2 Note: For CDT 1.2, the standard make project type and the new managed make project type have been separated out from the CDT "Platform".  When using the update site, please install the CDT feature as well as the Standard Build feature and optionally the Managed Make feature, in that order, or optionally selecting them and installing them at the same time (Process All). Other release notes for CDT 1.2 are available here.

CDT 2.0 Development Stream

The CDT 2.0 stream is currently under development. The objective of this stream is to keep up with the Eclipse 3.0 development and is targeted at release in sync with Eclipse 3.0, i.e. June 2004.

CDT 2.0 M6

Well, we've finally been able to figure out how to build properly (i.e. use Eclipse 2.1.2 to do the build). The CDT 2.0 M6 build is available from the following update site:

http://update.eclipse.org/tools/cdt/updates/builds/2.0M6

To install the update in Eclipse 3.0 M6, first uninstall any CDT version you happen to have currently installed. Then from the menu bar, select Help->Software Updates->Find and Install..., select “Search for new features to install”, add the update site with the above URL, and proceed through the rest of the wizard. As with 1.2, please make sure you install one or both builders to ensure you can create a C/C++ project (clicking on Select All works nicely).

Note that CDT 2.0 M6 will only run against Eclipse 3.0 M6 and later. This should hopefully be the last release where platform API changes blow us out of the water.

CDT 2.0 M5

The CDT 2.0 M5 build runs against Eclipse 3.0 M5. It does not run against earlier versions of Eclipse, or newer for that matter. A new M6 will be released to run against Ecilpse 3.0 M6. With the move to Eclipse 3.0, the CDT build scripts have failed. As a result, only the zip distributions are available. Simply unzip these into the eclipse install.

CDT 1.2.x Maintenance Stream

The CDT 1.2 stream is now in maintenance mode. A new release, 1.2.1 is expected in the first quarter of 2004. The 1.2 stream runs against versions 2.1.x of Eclipse. It does not run against Eclipse 3.0 builds.

CDT 1.2.1 RC 1

The CDT 1.2.1 RC 1 build can be found in it's own update site. This is the first candidate for the 1.2.1 maintenance release. This release fixes a number of problems found in the 1.2.0 GA release.

To install this build, remove any previous CDT 1.2.1 install you may have (i.e. all features and plugins named org.eclipse.cdt*). Then start up eclipse and use the following URL to create a Site Bookmark in the Update Manager.

http://update.eclipse.org/tools/cdt/updates/builds/1.2.1RC1

Add each feature you need to the Selected Updates list, then go to the list and Process All. You must install at least one of the builders (Standard or Managed or both) in order to create a C/C++ project.

Alternatively, you can download the zip file containing all of the CDT. Unzip this into the eclipse folder. This will install all of the platform specific features. However, Eclipse is smart enough to run only those features which are applicable to the current environment.

http://update.eclipse.org/tools/cdt/updates/builds/1.2.1RC1/cdt-1.2.1RC1.zip
