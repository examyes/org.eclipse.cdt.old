Contributed Plugins
CppUnit feature

The CppUnit feature is an integration of the CppUnit testing framework within the Eclipse CDT. CDT users can write CppUnit (JUnit-like) test cases in their C/C++ CDT projects, run them and view the results in a test result view. Further information on CppUnit is available at http://sourceforge.net/projects/cppunit

To use this plugin, you need to have the CppUnit framework installed locally on your computer. To do this, download CppUnit and install it from  http://prdownloads.sourceforge.net/cppunit

The CppUnit feature is available from the Eclipse CDT Update site at: http://update.eclipse.org/tools/cdt/updates/release
Eclipse-RPM

Eclipse-RPM is a graphical, Eclipse-based tool for packaging applications in the popular RPM Package Manager (RPM) format. Eclipse-RPM works alongside the CDT and aims to provide users with an easy way to package C/C++ applications in the RPM format.
Eclipse-OProfile

Eclipse-OProfile is a graphical, Eclipse-based front-end to the powerful OProfile profiling tool. Eclipse-OProfile is integrated with the CDT and aims to provide users with an easy-to-use interface for profiling C/C++ applications.
GNU Indent, formatter

This an example plugin illustrating how to integrate with the CDT formatter extension point.

  1. Unpack the zip-file
  2. make 'indent' available on your system:
    • under Linux go to http://www.gnu.org/software/indent/indent.html#TOCdownloading download the sources and compile it.
    • under Windows you can try the same, but it is much easier to use cygwin (you will find indent in the 'devel'-section. To make it work, make sure that your cygwin-bin-folder is in your PATH.
  3. Enable in the preference page; Preferences -> C/C++ -> Code Formater, choose in the drop menu "GNU Indent".
  4. Use ctl-shift-F or the the context menu to reformat selection or the entire file.
  5. See the GNU indent man to see how to set the options.

Manuals for GNU Tools
GCC docs zip GDB Docs zip make docs zip binutils doc zip

The default reference implementation of the CDT uses GNU Make, and GDB. The manuals for GNU Make, GCC, GDB and Bin Utils have been made available for inclusion within Eclipse. This Includes Make manual for Make v3.79.

Installation instructions:
Unpack the contents of the zip files into ECLIPSE_ROOT/plugins, where ECLIPSE_ROOT is the root of your Eclipse installation. The plugins are OS- independent and build-independent. Installing the plugins while Eclipse is running is harmless, but Eclipse will need to be restarted for the changes to take effect.
Compatability:
The plugins extend org.eclipse.help.toc, which is available in all currently distributed versions of Eclipse. The CDT is not required. The plugins may be installed in any desired combination.
